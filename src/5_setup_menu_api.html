Exercise: Set up the Menu API

Overview

In previous exercises, you have already set up a LittleLemon project that includes the Restaurant app.
You have also declared the Menu and Booking models and migrated them to the MySQL database.

In this exercise, you will build a REST API to perform CREATE, READ, UPDATE and DELETE operations on the Menu model.

Scenario

You will use Django REST Framework to build the APIs in the restaurant app.
You will also create the views using DRF's generic views.

1 Install Django REST Framework

Install the rest_framework package with PIP while remaining in the Django virtual environment directory.

pip install djangorestframework

Open the command prompt in the LittleLemon directory and start VS Code.

2 Add rest_framework to INSTALLED_APPS

Open settings.py file and add 'rest_framework' to the project's INSTALLED_APPS list:

INSTALLED_APPS = [
    ...
    'rest_framework',
    'restaurant',
    ...
]

3 Create MenuSerializer

In the restaurant app folder, create a new file called serializers.py and declare the MenuSerializer class:

# restaurant/serializers.py
from rest_framework import serializers
from .models import MenuItem

class MenuItemSerializer(serializers.ModelSerializer):
    class Meta:
        model = MenuItem
        fields = '__all__'          # or list specific fields: ['id', 'title', 'price', 'inventory']

4 Create API Views

Open restaurant/views.py and add the following classes:

from rest_framework import generics
from .models import MenuItem
from .serializers import MenuItemSerializer

class MenuItemsView(generics.ListCreateAPIView):
    queryset = MenuItem.objects.all()
    serializer_class = MenuItemSerializer

class SingleMenuItemView(generics.RetrieveUpdateDestroyAPIView):
    queryset = MenuItem.objects.all()
    serializer_class = MenuItemSerializer

5 Set up URL routing

restaurant/urls.py

from django.urls import path
from . import views

urlpatterns = [
    path('menu/', views.MenuItemsView.as_view(), name='menu-list'),
    path('menu//', views.SingleMenuItemView.as_view(), name='menu-detail'),
]

project urls.py (main LittleLemon/urls.py)

from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('restaurant/', include('restaurant.urls')),
]

6 Test the API

  1. Run the development server: 
    python manage.py runserver
  2. Open in browser: http://localhost:8000/restaurant/menu/

You should see the browsable API interface.
You can now:

  • GET – list all menu items
  • POST – create new menu item
  • GET / PUT / PATCH / DELETE on /restaurant/menu/1/ etc.
Success!
You have now created a complete CRUD REST API for the Menu using Django REST Framework generic views!
